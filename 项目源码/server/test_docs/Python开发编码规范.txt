Python开发编码规范

一、概述

本规范旨在统一公司内部Python项目的编码风格，提高代码可读性和可维护性。所有Python项目开发人员须遵循本规范进行代码编写。本规范主要参考PEP 8编码规范，并结合公司项目实际情况进行了补充和调整。

二、命名规范

2.1 模块和包命名
- 模块名使用小写字母，多个单词之间用下划线连接，如：user_service.py
- 包名使用小写字母，尽量不使用下划线，如：utils、models

2.2 类命名
- 使用大驼峰命名法（PascalCase），如：UserService、OrderManager
- 异常类以Error或Exception结尾，如：InvalidTokenError

2.3 函数和方法命名
- 使用小写字母和下划线，如：get_user_info、calculate_total_price
- 私有方法以单下划线开头，如：_validate_input
- 布尔类型的方法或变量建议使用is_、has_、can_前缀

2.4 变量命名
- 使用小写字母和下划线，如：user_name、order_count
- 常量使用全大写字母和下划线，如：MAX_RETRY_COUNT、DEFAULT_TIMEOUT
- 避免使用单字母变量名（循环变量i、j、k除外）

2.5 数据库相关命名
- 数据库名以db_开头
- 表名以t_开头，使用小写和下划线
- 字段名使用小写和下划线

三、代码格式

3.1 缩进
- 使用4个空格进行缩进，不使用Tab
- 续行应与打开定界符对齐

3.2 行长度
- 每行代码不超过120个字符
- 导入语句每行一个

3.3 空行
- 顶层函数和类定义之间空两行
- 类中的方法定义之间空一行
- 函数内部逻辑段落之间可空一行

3.4 导入顺序
- 标准库导入
- 第三方库导入
- 本地模块导入
- 各组之间用空行分隔

四、注释规范

4.1 文档字符串
- 所有公共模块、类、函数和方法都必须有文档字符串
- 使用三引号格式的文档字符串
- 文档字符串应说明功能、参数、返回值

4.2 行内注释
- 注释与代码之间至少间隔两个空格
- 注释使用中文编写，保持简洁明了
- 避免无意义的注释，如"# 定义变量"

五、异常处理

5.1 异常捕获
- 不要使用裸露的except语句，应指定具体的异常类型
- 异常处理中应记录日志
- 对外部服务调用必须进行异常捕获

5.2 异常抛出
- 自定义异常应继承自Exception或其子类
- 异常信息应清晰描述错误原因

六、日志规范

6.1 日志级别
- DEBUG：调试信息，仅在开发环境使用
- INFO：正常业务流程的关键节点信息
- WARNING：潜在问题的警告信息
- ERROR：错误信息，但程序可继续运行
- CRITICAL：严重错误，程序可能无法继续运行

6.2 日志内容
- 必须包含时间、日志级别、模块名、具体信息
- 敏感信息（如密码、Token）不得出现在日志中

七、安全规范

7.1 输入验证
- 所有外部输入必须进行验证和清理
- SQL查询必须使用参数化查询，禁止字符串拼接

7.2 密码处理
- 密码存储必须使用加密算法（如MD5、bcrypt）
- 禁止在代码中硬编码密码和密钥

八、版本要求

公司项目统一使用Python 3.10及以上版本，新项目优先使用Python 3.12。
