# 项目管理平台操作指南

## 一、平台简介

公司项目管理平台是基于敏捷开发理念打造的项目协作工具，支持项目立项、任务分配、进度跟踪、文档协作等功能。通过本平台，团队可以高效地管理项目全生命周期。

**平台访问地址：** http://pm.company.com

## 二、项目管理

### 2.1 创建项目

1. 点击首页右上角"新建项目"按钮
2. 填写项目基本信息：
   - **项目名称：** 必填，不超过50个字符
   - **项目编号：** 系统自动生成，格式为PJ-YYYYMMDD-XXX
   - **项目类型：** 产品研发、技术支持、运维保障等
   - **优先级：** 高、中、低
   - **起止日期：** 项目计划开始和结束日期
   - **项目描述：** 简要说明项目目标和范围
3. 添加项目成员并设置角色（项目经理、开发、测试、设计等）
4. 点击"创建"完成

### 2.2 项目设置

- **迭代管理：** 创建迭代周期，建议2-4周为一个迭代
- **标签管理：** 自定义标签用于任务分类，如"前端"、"后端"、"BUG"
- **通知设置：** 配置任务变更时的通知方式（站内消息、邮件）

## 三、任务管理

### 3.1 创建任务

在项目看板页面点击"+"号创建任务：

- **任务标题：** 简明扼要描述任务内容
- **任务类型：** 需求、任务、Bug、优化
- **优先级：** 紧急、高、中、低
- **指派人：** 选择负责该任务的成员
- **所属迭代：** 关联到对应的迭代周期
- **预计工时：** 预估完成所需的工作小时数
- **截止日期：** 任务完成的最后期限
- **描述：** 详细的任务描述，支持Markdown格式

### 3.2 任务状态流转

任务状态按以下流程流转：

- **待处理** → 任务创建后的初始状态
- **进行中** → 开发人员开始处理
- **已完成** → 开发完成，提交测试
- **测试中** → 测试人员进行验证
- **已验收** → 测试通过，产品验收
- **已关闭** → 任务最终完成

### 3.3 看板视图

项目看板以列的形式展示不同状态的任务：

- 支持拖拽任务卡片改变状态
- 可按指派人、优先级、标签筛选
- 任务卡片显示标题、指派人头像、优先级标识、截止日期

### 3.4 甘特图视图

- 以时间轴的形式展示任务的起止时间和进度
- 支持拖拽调整任务时间
- 红色标识已延期的任务

## 四、文档协作

### 4.1 项目文档

每个项目有独立的文档空间：

- **需求文档：** 记录项目需求和功能说明
- **设计文档：** 技术方案、架构设计等
- **会议纪要：** 项目会议的记录
- **上线文档：** 部署方案、上线检查清单

### 4.2 在线编辑

- 支持多人同时在线编辑文档
- 自动保存，支持版本回溯
- 支持Markdown和富文本两种编辑模式

## 七、常见问题

**Q1: 如何批量导入任务？**
A1: 在任务列表页面点击"导入"，下载Excel模板，按模板格式填写任务信息后上传导入。

**Q2: 如何设置任务的依赖关系？**
A2: 在任务详情页，点击"添加关联"，选择前置任务或后置任务，系统会自动校验时间冲突。

**Q3: 如何导出项目报表？**
A3: 在统计报表页面，选择需要的报表类型和时间范围，点击右上角"导出"按钮，支持导出为Excel和PDF格式。

**Q4: 项目归档后还能查看吗？**
A4: 已归档的项目可在"归档项目"中查看，数据保留但不可修改。需要重新激活的，请联系项目管理部门。
